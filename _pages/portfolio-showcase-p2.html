Harmony Design System (Bot Behaviour Design)

  • Low-code platform that creates features and builds a base application for businesses
  • Multiple types of end-product users
  • Constant product iteration from user quantitative feedback
  • Balancing as a Product Designer and SCSS/HTML development

Understand

  • User types: developers that require fast repetitive building blocks and features specifically for their business requirements
  • Features had to be usable by any type of business
  • Have base requirements for features but also allow flexibility
  • 4 behaviours required: Timelines, CRUD, Forms and Workflows

Research & Test

  • Find common patterns and user experiences from other industries
  • The ideal user journeys for each feature were created
  • Different types of workshops were performed depending on complexity of the feature
  • Sketches and user journeys were refined from user feedback
  • Wireframes help nailed the ideal user experience and feel for the feature

Design and development

  • Design system was created, and mainatined
  • A SCSS styling library was created for these features
  • These components were consistently refined from user feedback once the features have been deployed
  • Given WorkingMouse were using these bots everyday for client-facing work, I became the company's styling the bot mentor for multiple developers
